# Team roles

Each character has a main role. This is what they are best at.

## Analyst

The analyst clarifies work and raises task clarity.

A clear task is faster to finish and less likely to become a bad release. But analysts get tired too, so analyzing everything is not always the best move.

## Backend

Backend is the server side: APIs, databases, payments, integrations, and background jobs.

Weak backend work can create bugs in data, payments, sync, or reports.

## Frontend

Frontend is what the user sees: pages, forms, buttons, and interface states.

Weak frontend work can break user flows: a form does not submit, a screen freezes, or data is displayed incorrectly.

## QA

QA finds bugs and checks that changes actually work.

QA does not build the feature instead of a developer. QA lowers risk, finds problems, and can turn bugs into new rework subtasks.

## SRE

SRE protects production reliability: monitoring, failover, load, and incident scenarios.

If a task can hit production hard, SRE reduces the blast radius. Without SRE, a dirty release can hurt trust and clients much more.

## Outsource

Outsource is an expensive fallback when the right person is missing or exhausted.

It can save a stuck task, but it costs budget and should not replace planning.
